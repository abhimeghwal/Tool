How to Fix This Issue

This error occurs when GitHub Pages hasn't been properly configured. Follow these steps to set up your site:

1

Select Publishing Source

Go to your repository on GitHub, then navigate to:

Settings → Pages

Under "Source", select a branch (usually main or gh-pages) and folder (/root or /docs).

2

Add index.html

Your repository must contain an index.html file at the root of the selected branch and folder.

<!-- Basic index.html example -->
<!DOCTYPE html>
<html>
<head>
  <title>My Site</title>
</head>
<body>
  <h1>Hello World!</h1>
</body>
</html>
3

Check Repository Structure

For user/organization sites, your repository must be named:

<username>.github.io

For project sites, you can use any repository name, but must configure the source branch in settings.

Troubleshooting Tips

1

Check Build Status

GitHub Pages sites can take up to 10 minutes to build after you push changes. Check the build status in:

Settings → Pages → Build and deployment

2

Verify File Location

Ensure your index.html is in the root of the selected branch or in the docs folder if you selected that option.

Case sensitivity matters: index.html not Index.HTML

3

Custom Domain Setup

If using a custom domain, verify:

  • Your DNS settings point to GitHub
  • You have a CNAME file in your repository
  • HTTPS is properly configured
